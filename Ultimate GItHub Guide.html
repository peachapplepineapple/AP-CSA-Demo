MOD 01

Why GitHub

Imagine writing a 500-line Java program. You make changes Monday that feel great — but Thursday you realize Monday's version was better. Without GitHub, that code is gone forever. GitHub stores every version you have ever saved and lets you go back to any of them at any time.

🎯
Think of it as Google Docs version history, but for code.

The Four Pillars

#PillarWhat It Means
1Version ControlEvery save (commit) is a snapshot. Roll back to any of them — like a game checkpoint.
2CollaborationMultiple people work on the same project at the same time. Git merges everyone's work.
3Free CodeGitHub hosts millions of open-source projects. Download and learn from any of them.
4Real WorkflowCompanies separate front-end and back-end into different repos. Today you work exactly like that.

Vocabulary

WordDefinition
RepositoryA folder GitHub tracks. All project files live here.
CommitA saved snapshot with a description.
PushSend your local commits up to GitHub.
PullDownload the latest changes from GitHub.
CloneCopy a GitHub repo to your laptop for the first time.
ForkYour personal copy of someone else's repo on GitHub.
Pull RequestA request to merge your changes into someone else's repo.
OriginThe nickname Git uses for your GitHub URL.
RemoteAny version of your repo stored outside your laptop.
BranchA parallel version of your code for safe experimentation.
MOD 02

SSH Setup

SSH (Secure Shell) proves to GitHub that your laptop is really you. Without it you would type your password every single push. SSH replaces that permanently — and it is far more secure.

🏠
Your laptop holds the private key. GitHub holds the public key (the lock). Only your laptop can open the door.

2.1 — Create a GitHub Account

1
Go to github.com and click Sign Up
2
Use your personal email address
3
Verify your email by clicking the link GitHub sends
4
Choose the free plan
5
Pick a professional username — it appears on every commit you make

2.2 — Generate Your SSH Key

Open Terminal (Mac/Linux) or Git Bash (Windows).

Check if keys already exist:

ls -al ~/.ssh

If you see id_ed25519 and id_ed25519.pub — skip to 2.3. Otherwise continue.

Generate a new key (replace with your GitHub email):

ssh-keygen -t ed25519 -C "your_email@example.com"
FlagWhat It Does
-t ed25519Uses the modern, secure ed25519 algorithm
-C "email"Labels the key with your account email
💡
When asked "Enter file to save the key" — press Enter to use the default. When asked for a passphrase — press Enter twice to skip.

Start the SSH agent:

eval "$(ssh-agent -s)"

You should see Agent pid 12345 — the number will differ, that is fine.

Add your key to the agent:

ssh-add ~/.ssh/id_ed25519

Copy your public key:

# Mac
pbcopy < ~/.ssh/id_ed25519.pub

# Windows (Git Bash)
clip < ~/.ssh/id_ed25519.pub

# Linux
cat ~/.ssh/id_ed25519.pub
# Manually select and copy the full output
⚠️
Never share id_ed25519 (no .pub). That is your private key — treat it like a password. Only share the .pub version.

2.3 — Add the Key to GitHub

1
GitHub.com → click your profile photo → Settings
2
Left sidebar → SSH and GPG keys
3
Click New SSH key
4
Title: My School MacBook (or whatever device this is)
5
Key type: Authentication Key
6
Key: paste the public key (starts with ssh-ed25519)
7
Click Add SSH key

2.4 — Test the Connection

ssh -T git@github.com

Success looks like:

Hi yourusername! You've successfully authenticated,
but GitHub does not provide shell access.
ℹ️
If you see "Are you sure you want to continue connecting (yes/no)?" — type yes and press Enter. This only appears the very first time and is completely normal.

The Class Project

In BlueJ: File → New Project → name it ap-csa-demo. Create these two classes inside the project.

StudentProfile.java

/**
 * StudentProfile — represents a student in AP CSA.
 * Everyone adds themselves to practice GitHub collaboration.
 *
 * @author  Your Name Here
 * @version 1.0
 */
public class StudentProfile {

    private String firstName;
    private String lastName;
    private int    gradeLevel;
    private String favLanguage;
    private String funFact;

    public StudentProfile(String firstName, String lastName,
                          int gradeLevel, String favLanguage,
                          String funFact) {
        this.firstName   = firstName;
        this.lastName    = lastName;
        this.gradeLevel  = gradeLevel;
        this.favLanguage = favLanguage;
        this.funFact     = funFact;
    }

    public String getFirstName()   { return firstName; }
    public String getLastName()    { return lastName; }
    public int    getGradeLevel()  { return gradeLevel; }
    public String getFavLanguage() { return favLanguage; }
    public String getFunFact()     { return funFact; }

    public void setFavLanguage(String newLang) { this.favLanguage = newLang; }
    public void setFunFact(String newFact)     { this.funFact = newFact; }

    public String introduce() {
        return "=================================\n" +
               "  Name      : " + firstName + " " + lastName + "\n" +
               "  Grade     : " + gradeLevel + "\n" +
               "  Fav Lang  : " + favLanguage + "\n" +
               "  Fun Fact  : " + funFact + "\n" +
               "=================================";
    }

    public boolean sameGrade(StudentProfile other) {
        return this.gradeLevel == other.gradeLevel;
    }

    @Override
    public String toString() {
        return firstName + " " + lastName + " (Grade " + gradeLevel + ")";
    }
}

ClassRoster.java

import java.util.ArrayList;

/**
 * ClassRoster — stores all StudentProfile objects for AP CSA.
 *
 * @author  Your Name Here
 * @version 1.0
 */
public class ClassRoster {

    private ArrayList<StudentProfile> roster;
    private String className;

    public ClassRoster(String className) {
        this.className = className;
        this.roster    = new ArrayList<>();
    }

    public void addStudent(StudentProfile student) {
        roster.add(student);
        System.out.println("Added: " + student);
    }

    public void printAll() {
        System.out.println("\n=== " + className + " Roster ===");
        for (StudentProfile s : roster) {
            System.out.println(s.introduce());
            System.out.println();
        }
    }

    public int size() { return roster.size(); }

    public StudentProfile findByLastName(String lastName) {
        for (StudentProfile s : roster) {
            if (s.getLastName().equalsIgnoreCase(lastName)) return s;
        }
        return null;
    }

    public ArrayList<StudentProfile> getByGrade(int grade) {
        ArrayList<StudentProfile> result = new ArrayList<>();
        for (StudentProfile s : roster)
            if (s.getGradeLevel() == grade) result.add(s);
        return result;
    }
}

BlueJ's built-in window only shows Java output while a program runs — it cannot run Git commands. You need your system terminal.

  • Mac: Command + Space → type Terminal → Enter
  • Windows: Windows key → type Git Bash → Enter

cd ~/Documents
ls

Find your project name in the output, then enter it:

cd ap-csa-demo
ls

You should see StudentProfile.java and ClassRoster.java listed.

💡
Not sure where BlueJ saved your project? Look at the BlueJ title bar — it shows the full file path. Then run cd /that/exact/path

5.3 — Configure Git — One-Time Setup

Tell Git who you are. This name and email appear on every commit you make.

git config --global user.name "Your Full Name"
git config --global user.email "youremail@example.com"

Verify it saved:

git config --global user.name
git config --global user.email

Your name and email should print back. If they do, move on.

user.nameThe name shown on your commits
user.emailMust match your GitHub account email

5.5 — Initialize Git in Your Project

Back in terminal, inside your ap-csa-demo folder:

git init

Expected output:

Initialized empty Git repository in /Users/you/Documents/ap-csa-demo/.git/

Git created a hidden .git folder. That folder tracks all your changes from this point forward.

5.6 — Connect Your Folder to GitHub

git remote add origin git@github.com:YOURUSERNAME/ap-csa-demo.git

Replace YOURUSERNAME with your actual GitHub username. Use the exact URL from the GitHub setup page.

PartWhat It Means
git remote addConnect this folder to a location on the internet
originThe nickname for that location — everyone uses this name by convention
git@github.com:...The SSH address of your GitHub repo

Confirm it worked:

git remote -v

Expected output:

origin  git@github.com:yourusername/ap-csa-demo.git (fetch)
origin  git@github.com:yourusername/ap-csa-demo.git (push)

5.7 — First Push — Upload Your Code

Run these five commands in order:

Stage all files:

git add .

The . means everything in this folder. Nothing is printed — that is normal.

Check what was staged:

git status

Your .java files should appear in green under "Changes to be committed." If they are red, run git add . again.

Commit — save the snapshot:

git commit -m "Initial commit - add StudentProfile and ClassRoster"

Always write a real message. The -m flag is where the message goes.

Name your branch:

git branch -M main

GitHub uses main as the default branch name. This renames your branch to match.

Push to GitHub:

git push -u origin main

After this, future pushes only need git push. Go to your GitHub repo page and refresh — your .java files are now live.

🚨
Push Failing? Port 22 Blocked — Fix It Here

If your push gives you this error:

Received disconnect from 140.82.112.4 port 22: Connection blocked
fatal: Could not read from remote repository.

Your network is blocking port 22 (the default SSH port). This is very common on school and office Wi-Fi. The fix is to route SSH through port 443 — the same port as normal websites, which is never blocked.

Step 1 — Open your SSH config file:

nano ~/.ssh/config

If the file does not exist yet, nano creates it automatically.

Step 2 — Paste this exactly:

Host github.com
    Hostname ssh.github.com
    Port 443
    User git
LineWhat It Does
Host github.comApply this rule whenever connecting to github.com
Hostname ssh.github.comRedirect to GitHub's alternate SSH server
Port 443Use the HTTPS port — always open on any network
User gitGitHub always uses git as the SSH username

Step 3 — Save and exit nano:

Press Control + X then Y then Enter

Step 4 — Test the connection:

ssh -T git@github.com

Expected output:

Hi yourusername! You've successfully authenticated,
but GitHub does not provide shell access.

Step 5 — Push again:

git push -u origin main
✅
The config file is permanent — you never need to do this again on this machine. Many school networks block port 22 as a security policy. GitHub provides an alternate SSH server on port 443 specifically for this situation.

5.8 — Daily Workflow — The Git Loop

Every time you write code:

git add .
git commit -m "Describe what you changed"
git pull
git push
Write code
|
v
git add . select changed files
|
v
git commit -m "..." save a labeled snapshot
|
v
git pull get any updates from GitHub FIRST
|
v

⚠️
Always git pull before git push. If a classmate pushed changes since your last pull, Git will reject your push until you pull their changes first.

5.9 — Check Commands

git status        # what files changed since last commit?
git log --oneline # compact list of all past commits
git diff          # show exactly which lines changed

7.1 — Fork a Classmate's Repo

1
Go to github.com/CLASSMATE-USERNAME/ap-csa-demo
2
Click Fork (top right)
3
Select your own account as the destination
4
GitHub copies the repo — the URL becomes github.com/YOUR-USERNAME/ap-csa-demo

7.2 — Clone YOUR Fork

cd ~/Documents
git clone git@github.com:YOUR-USERNAME/ap-csa-demo.git
cd ap-csa-demo

Clone your fork, not the original. Your fork's URL has your username in it.

7.3 — Sync Your Fork

If the original gets new commits after you forked, your fork falls behind:

# Add the original as a second remote (do this once)
git remote add upstream git@github.com:CLASSMATE-USERNAME/ap-csa-demo.git

# Pull their latest changes
git fetch upstream
git merge upstream/main

# Push the synced version to your fork
git push

upstream = the original repo you forked from. origin = your fork.

7.4 — Send a Pull Request

After making your changes:

git add .
git commit -m "Add compareGrade() to StudentProfile"
git push
1
Go to your fork: github.com/YOUR-USERNAME/ap-csa-demo
2
Click Contribute → Open pull request
3
Review the diff GitHub shows you
4
Add a title and description explaining what you added and why
5
Click Create pull request
MOD 08

Challenges & Answers

⭐ Level 1 — Challenge A
compareGrade()
Add a compareGrade() method to StudentProfile that returns a String saying whether you are in a higher, lower, or same grade as another student.
⭐ Level 1 — Challenge B
getFullName()
Add a getFullName() method that returns the student's full name as one String — for example "Maria Santos".

Answers

1A compareGrade() 
public String compareGrade(StudentProfile other) {
    if (this.gradeLevel > other.gradeLevel) {
        return this.firstName + " is in a higher grade than " + other.firstName;
    } else if (this.gradeLevel < other.gradeLevel) {
        return this.firstName + " is in a lower grade than " + other.firstName;
    } else {
        return this.firstName + " and " + other.firstName + " are in the same grade!";
    }
}
1B getFullName() 
public String getFullName() {
    return firstName + " " + lastName;
}

MOD 10

Errors & Fixes

Permission denied (publickey)

Your SSH key is not recognized by GitHub.

ssh-add ~/.ssh/id_ed25519
ssh -T git@github.com

If it still fails, redo Module 2 from the beginning.

fatal: not a git repository

You are running Git in the wrong folder.

pwd            # shows where you are now
ls             # shows what files are here
cd ap-csa-demo # go to the correct folder

error: remote origin already exists

You already ran git remote add origin once before.

git remote remove origin
git remote add origin git@github.com:YOURUSERNAME/ap-csa-demo.git

ERROR: Repository not found

The SSH URL is wrong, or you do not have access to that repo.

1
Go to your GitHub repo → green Code button → SSH
2
Copy the exact URL
3
Run git remote remove origin
4
Run git remote add origin PASTE-YOUR-URL-HERE

Push rejected (non-fast-forward)

Someone else pushed after your last pull. Get their changes first.

git pull
git push

Connection blocked on port 22

See the full fix in Module 5. Short version:

nano ~/.ssh/config

Paste:

Host github.com
    Hostname ssh.github.com
    Port 443
    User git

Save: Control + X → Y → Enter. Then push again.

Merge Conflict

Two people edited the same file. Open the conflicted file and find:

<<<<<<< HEAD
your version of the code
=======
their version of the code
>>>>>>> main

Keep the version you want. Delete the other version and all three marker lines. Then:

git add .
git commit -m "Resolve merge conflict"
git push

Reset Git and Start Over

⚠️
Only use this if everything is broken and you want a completely clean start.
rm -rf .git
git init
git remote add origin git@github.com:YOURUSERNAME/ap-csa-demo.git
git add .
git commit -m "Fresh start"
git branch -M main
git push -u origin main --force
⚠️
--force rewrites history on GitHub. Only use it on your own personal repo — never on a shared class repo.

How SSH Works

| (id_ed25519) | (pasted in Settings)
| |
| "Can I connect?" ------------------> |
| | checks keys match
| <--------- "Yes, verified." |
| |
| git push (your code) --------------> |

The private key never leaves your laptop. GitHub only ever sees the public key.

| |

|
the nickname you give that URL (everyone uses "origin")
